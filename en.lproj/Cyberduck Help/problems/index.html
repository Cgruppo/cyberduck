Known Problems

The application doesn't launch; it just bounces a few times in the dock.

Make sure to install the latest Java Update available in Software Update.

My preferred editor isn't supported.

An editor must support the ODB external editor protocol to be added.

Special characters such as Umlaute aren't displayed correctly in the browser.

Try to change the character encoding used. See 'View' > 'Encoding'.

The folder size isn't displayed correctly.

This is not the size of its content but the size of the folder itself.

When listing a directory on a FTP server I get the error message in response to the 'LIST -a' command: 550 Directory not found.

Uncheck the option 'FTP > Send LIST -a command' in the Preferences.

The server does only allow one single connection; I can't transfer any files!

Limit the maximum number of simultaneous to a server to one in the 'Advanced' tab in the Preferences.

The connection to the server frequently times out; I want it to stay it alive if at all possible!

You can force Cyberduck to send a no operation command at a given time interval. See the 'Advanced' tab in the Preferences.

I don't want to synchronize files if only e.g. the minutes differ in the timestamp!

You can configure Cberduck to ignore certain values when comparing the modification date of files. Open a Terminal.app and type:

defaults write ch.sudo.cyberduck queue.sync.ignore.second true
defaults write ch.sudo.cyberduck queue.sync.ignore.minute true
defaults write ch.sudo.cyberduck queue.sync.ignore.hour false

By default, milliseconds and seconds are ignored.
