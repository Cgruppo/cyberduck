Cyberduck Help
Common Problems

Special characters such as Umlaute aren't displayed correctly in the browser.
Try to change the character encoding used. See 'View' > 'Encoding'.

The folder size isn't displayed correctly.
This is not the size of its content but the size of the folder itself.

When listing a directory on a FTP server I get the error message in response to the 'LIST -a' command: 550 Directory not found.
Uncheck the option 'FTP > Send LIST -a command' in the Preferences.

It is not possible to change permissions. The error message 'FTP Error: SITE not understood' or similar is displayed.
The server does not support this feature (which is an optional extension to the FTP protocol) and can not be used.

The server does only allow one single connection; I can't transfer any files!
Limit the maximum number of simultaneous to a server to one in the 'Advanced' tab in the Preferences. The transfer of a file will then wait until you close the browsing connection. If the browsing connection should be closed automatically, then also check 'Force other connections to close when maximum is reached'.

The connection to the server frequently times out; I want it to stay it alive if at all possible!
You can force Cyberduck to send a no operation command at a given time interval. See the 'Advanced' tab in the Preferences.

I often get an error titled IO Error: Broken pipe.
There was a problem with the unerlying network stream either because the connection was too slow or the server wouldn't give a response.

I get an error titled IO Error: Read Timed Out when attempting to open a connection.
There is either a firewall blocking requests or no server is listening at the given address and protocol.

I can't select any external editor.
Install one of the supported editors and launch it first. Also relaunch Cyberduck if the editor is still not selectable. If that still doesn't work out, please refer to this article.

My preferred editor isn't supported.
An editor must support the external editor protocol (ODB).

I want ot use the latest beta of skEdit which supports the external editor protocol as an editor.
Issue the following commands in a Terminal.app window (copy & paste) and relaunch Cyberduck:

defaults write ch.sudo.cyberduck editor.name skEdit
defaults write ch.sudo.cyberduck editor.bundleIdentifier org.skti.skEdit

I don't want to synchronize files if only e.g. the minutes differ in the timestamp!
You can configure Cberduck to ignore certain values when comparing the modification date of files. Open a Terminal.app and type:

defaults write ch.sudo.cyberduck queue.sync.ignore.second true
defaults write ch.sudo.cyberduck queue.sync.ignore.minute true
defaults write ch.sudo.cyberduck queue.sync.ignore.hour false

By default, milliseconds and seconds are ignored.
