Aide de Cyberduck
Problèmes connus

Des caractères spéciaux comme les trémas ne s'affichent pas correctement dans le navigateur.
Essayez de changer l'encodage de caractères utilisé. Voyez “Présentation” > “Encodage”. Essayez UTF-8 (par défaut), ISO-8859-1 et Windows-1252.

La taille du dossier ne s'affiche pas correctement.
Ce n'est pas la taille de son contenu mais celle du dossier lui-même.

Lors du listage d'un dossier sur un serveur FTP j'obtiens ce message d'erreur en réponse à la commande 'LIST -a' : 550 Directory not found.
Décochez l'option “FTP” > “Envoyer la commande 'LIST -a'” dans les Préférences.

Il n'est pas possible de changer les permissions. Le message d'erreur 'FTP Error: SITE not understood' ou similaire s'affiche.
Le serveur ne supporte pas cette caractéristique (qui est une extension optionnelle du protocole FTP). Elle ne peut pas être utilisée.

Le serveur n'autorise qu'une seule connexion ; je ne peux transférer aucun fichier !
Limitez le “Nombre maximal de connexions simultanées à un serveur” dans l'onglet “Avancé” des Préférences. Le transfert d'un fichier attendra alors que vous vous déconnectiez. Si la connexion doit être fermée automatiquement, cochez aussi “Forcer la fermeture des autres connexions quand le maximum est atteint”.

La connexion s'interrompt fréquemment ; si c'est possible, j'aimerais qu'elle reste active !
Vous pouvez obliger Cyberduck à “Ne pas envoyer d'opération” à un interval régulier. Voyez l'onglet “Avancé” dans les Préférences.

J'obtiens souvent une erreur intitulée IO Error: Broken pipe.
Un problème est survenu dans l'accès au réseau parce que soit la connexion était trop lente, soit le serveur ne répondait pas. Vérifiez aussi le câble, le routeur…

J'obtiens une erreur intitulée IO Error: Read Timed Out en essayant d'ouvrir une connexion.
Soit un firewall bloque les requêtes, soit aucun serveur n'est actif à l'adresse et par le protocole indiqués.

Je ne peux choisir aucun éditeur externe.
Installez un éditeur compatible et lancez-le d'abord. Si aucun éditeur ne peut encore être sélectionné, relancer aussi Cyberduck. Si ça ne marche toujours pas, consultez  cet article.

Mon éditeur préféré n'est pas accepté.
Un éditeur doit pouvoir supporter le protocole pour éditeur externe (ODB).

I want ot use the latest beta of skEdit which supports the external editor protocol as an editor.
Issue the following commands in a Terminal.app window (copy & paste) and relaunch Cyberduck:

defaults write ch.sudo.cyberduck editor.name skEdit
defaults write ch.sudo.cyberduck editor.bundleIdentifier org.skti.skEdit

-->

Je ne veux pas synchroniser de fichiers si, p. ex., seules les minutes diffèrent  dans l'heure de modification !
Vous pouvez configurer Cyberduck pour qu'il ignore certaines valeurs lorsqu'il compare les dates et heures de modification des fichiers. Lancez le Terminal et tapez :

defaults write ch.sudo.cyberduck queue.sync.ignore.second true
defaults write ch.sudo.cyberduck queue.sync.ignore.minute true
defaults write ch.sudo.cyberduck queue.sync.ignore.hour false

Par défaut, les millisecondes et les secondes ne sont pas prises en compte.
